# 宗门总览页面布局设计

## 一、设计原则

1. **核心信息突出**：优先展示宗门核心数据和状态
2. **信息模块化**：将相关信息分组展示，便于用户快速获取
3. **交互友好**：提供明确的操作选项和状态反馈
4. **视觉层次清晰**：通过边框、分隔线和缩进区分不同功能区域
5. **响应式设计**：支持动态布局调整，适应不同屏幕尺寸
6. **快捷键支持**：提供常用操作的快捷键，提升操作效率

## 二、页面布局结构

## 三、区域功能详细说明

### 1. 顶部导航栏

**位置**：界面顶部，3行×160字符
**功能**：显示核心游戏信息，提供快速访问和全局搜索
**详细内容**：
- 第一行：宗门名称、地点、天气、游戏时间、游戏速度、全局搜索框
- 第二行：宗门关键数据（弟子总数、资源、设施、占领区域等）
- 第三行：功能分类导航（核心管理、功能区域、生产系统、系统）

### 2. 左侧菜单栏

**位置**：界面左侧，52行×30字符
**功能**：提供游戏主要功能的快速导航
**详细内容**：
- 核心管理：宗门总览（当前选中）、弟子管理、资源管理、设施建设
- 功能区域：地图探索、任务大厅、功法堂
- 生产系统：炼丹房、炼器房
- 系统：社交、设置
- 使用数字快捷键1-0,S快速访问

### 3. 中央内容区

**位置**：界面中央，34行×100字符
**功能**：展示宗门总览详细内容
**详细内容**：

#### 3.1 宗门基本信息
- 宗门名称、地点、等级、类型
- 创建时间、稳定度、发展趋势

#### 3.2 核心数据统计
- 弟子总数、灵石储备、设施数量、占领区域
- 贡献总量、各境界弟子数量
- 表格形式展示，清晰直观

#### 3.3 宗门影响力
- 地区影响力、声望上限
- 友好宗门、敌对宗门

#### 3.4 核心设施状态
- 核心建筑（青灵殿）状态和加成
- 生产设施（灵田、炼丹房、炼器阁）状态和产出
- 设施等级、加成效果、当前状态

#### 3.5 近期动态
- 弟子突破、资源变化、新弟子报名、使者来访等
- 滚动显示最新事件

#### 3.6 发展建议
- 根据宗门当前状态，提供智能发展建议
- 优先级标记，引导用户决策

#### 3.7 功能标签页
- 宗门概览（当前选中）、发展趋势、影响力分析、事件记录、设施管理
- 支持Tab键切换

### 4. 右侧信息区

**位置**：界面右侧，15行×30字符
**功能**：显示掌门信息和快捷操作
**详细内容**：

#### 4.1 掌门信息
- 掌门名称、境界
- 气血、灵力、寿元、贡献
- 状态加成

#### 4.2 宗门快捷操作
- 核心管理功能快捷入口
- 生产系统功能快捷入口

#### 4.3 当前状态加成
- 设施加成、丹药加成等

#### 4.4 宗门当前状态
- 运行状态、发展阶段
- 在线弟子数量、AI活跃数量

#### 4.5 基础快捷操作
- 搜索、帮助、设置、菜单等

### 5. 底部状态栏

**位置**：界面底部，5行×160字符
**功能**：显示系统信息和快捷键提示
**详细内容**：
- 第一行：新任务、弟子传讯、宗门事件、自动存档状态、帮助、菜单、当前模式、游戏速度、运行时间、版本号
- 第二行：快捷键提示
- 第三行：消息提示
- 第四行：系统状态（内存使用、CPU使用率、网络、存档、日志、音效、音乐）
- 第五行：开发者信息

## 八、实现建议

1. **模块化开发**：将中央内容区的各个模块作为独立组件开发
2. **状态管理**：使用状态管理库管理界面状态
3. **响应式布局**：实现动态布局调整
4. **快捷键支持**：实现完整的快捷键系统
5. **性能优化**：优化长列表滚动性能
6. **测试**：进行多场景测试，确保布局在不同情况下正常显示
7. **用户反馈**：收集用户反馈，持续优化布局
