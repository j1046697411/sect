# 弟子招募系统设计

## 基本信息
- 状态：已确认
- 优先级：P1
- 涉及系统：角色系统

## 需求描述
设计弟子招募机制，通过固定周期自动招募新弟子，由长老主持，根据宗门等级、规模和长老能力动态计算招募人数。

## 设计决策

| 维度 | 决策 | 说明 |
|------|------|------|
| 触发方式 | 固定周期 | 可配置，如3年/5年一次 |
| 招募人数 | 动态计算 | 宗门等级/规模 + 主持长老能力 |
| 资质决定 | 声望影响 | 根据宗门声望影响弟子资质 |
| 招募成本 | 多资源消耗 | 消耗灵石和食物 |
| 主持角色 | 长老主持 | 长老负责主持招募活动 |
| 长老状态 | 后台模式 | 不影响长老执行其他任务 |

## 用户故事
作为宗门管理者，我希望每隔固定周期自动招募新弟子，以便补充宗门人才储备。

## 功能要点
- [ ] 招募周期配置（默认值：3年）
- [ ] 招募人数计算公式（宗门等级 + 规模 + 长老能力）
- [ ] 弟子资质随机生成（受宗门声望影响）
- [ ] 资源消耗计算（灵石 + 食物）
- [ ] 长老分配机制
- [ ] 招募事件通知

## 验收标准
- [ ] 招募周期可配置
- [ ] 招募人数根据多维因素动态计算
- [ ] 弟子资质分布合理
- [ ] 资源消耗正确扣除
- [ ] 长老主持期间不影响其他任务
