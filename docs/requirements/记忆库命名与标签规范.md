## 一、标题命名规范

### 命名原则
1. **简洁** - 控制在 10 字以内
2. **可读** - 纯中文，避免中英混杂
3. **可搜索** - 关键词前置
4. **去冗余** - 目录已提供分类，标题不再重复

### 命名格式

```
{主题}.md           # 通用格式
{系统}-{主题}.md    # 需要区分系统时
```

### 改名示例

| 原标题 | 新标题 | 理由 |
|--------|--------|------|
| `角色系统设计角色层级系统.md` | `角色层级.md` | 去冗余 |
| `战斗系统设计战斗机制与类型.md` | `战斗机制.md` | 简洁 |
| `游戏可玩性设计ai行为信息分级.md` | `信息分级.md` | 去掉系统前缀 |
| `ecs-query-性能优化.md` | `Query性能优化.md` | 纯中文 |
| `用户偏好代码风格.md` | `代码风格.md` | 目录已是 preferences |
| `外门弟子晋升选拔机制-任务制.md` | `晋升机制-任务制.md` | 去掉"选拔"冗余 |

## 二、标签体系

### 标签层级

```
一级标签（系统域）    二级标签（技术栈）    三级标签（类型）
─────────────────    ─────────────────    ─────────────────
#角色系统             #ecs                 #规范
#经济系统             #kotlin              #方案
#战斗系统             #compose             #陷阱
#功法系统             #测试                #偏好
#设施系统                                  #性能
#可玩性
```

### 标签规则

1. **必选标签**：至少一个系统域标签
2. **可选标签**：技术栈 + 类型标签
3. **格式统一**：`#标签` 无空格

### 标签示例

```markdown
## 角色层级

> 标签: #角色系统 #规范

## Query性能优化

> 标签: #ecs #性能 #方案

## Compose初始化陷阱

> 标签: #compose #陷阱
```
