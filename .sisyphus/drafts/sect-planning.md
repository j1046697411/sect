## 用户确认的需求
1. **开发方式**: 敏捷迭代 - 先MVP后扩展
2. **规划范围**: 完整14个子系统规划
3. **使用现有**: ECS框架实现业务逻辑

## 需求分析总结

### 14个子系统
1. 角色系统（弟子/内门/亲传/长老/掌门）
2. 修炼系统（6大境界 + 功法 + 突破）
3. 设施系统
4. 资源系统
5. 俸禄系统
6. 任务系统
7. 战斗系统
8. 经济与交易系统
9. 事件系统
10. AI决策机制
11. 观察系统

### 核心游戏循环
```
制定政策 → AI执行 → 观察反馈 → 事件决策 → 调整政策 → [循环]
```

### 关键设计决策
- 宏观管理视角（不做微观操作）
- AI自主运转（所有角色根据职位自动处理事务）
- 世代更替（老角色逝去，新角色成长）
- 时间系统：1年 = 现实6分钟

### 结论
这是一个**从零开始实现业务逻辑的项目**。ECS框架是成熟的基础设施，14个子系统需要全部从头构建。

### MVP范围建议
基于敏捷迭代原则，建议MVP包含：
1. 角色系统（弟子创建、属性、基本状态）
2. 修炼系统（境界、修炼、突破）
3. 基础时间系统（游戏循环）
4. 简单观察系统（查看弟子）

暂不包含：
- 战斗系统、任务系统（可后期迭代）
- 经济交易系统
- 完整AI决策系统
