# OpenSkill 任务和技能清单

## 1. algorithmic-art

**描述**：使用 p5.js 创建算法艺术，支持种子随机性和交互式参数探索。

**功能**：
- 创建算法艺术哲学（.md 文件）
- 生成 p5.js 交互式艺术作品（.html + .js 文件）
- 支持种子随机化和参数调整

**使用场景**：
- 用户请求使用代码创建艺术
- 生成艺术、算法艺术、流场或粒子系统
- 需要创建原创算法艺术

**实现方式**：
- 两步流程：先创建算法哲学，再实现 p5.js 代码
- 使用模板文件 `templates/viewer.html` 作为基础
- 支持交互式参数调整

## 2. brand-guidelines

**描述**：将 Anthropic 的官方品牌颜色和排版应用到任何可能受益于 Anthropic 外观和风格的工件上。

**功能**：
- 应用 Anthropic 品牌颜色
- 应用 Anthropic 品牌字体
- 确保视觉一致性

**使用场景**：
- 需要应用品牌颜色或风格指南
- 需要视觉格式化或公司设计标准

**实现方式**：
- 使用指定的品牌颜色：Dark (#141413)、Light (#faf9f5)、Mid Gray (#b0aea5)、Light Gray (#e8e6dc)
- 强调色：Orange (#d97757)、Blue (#6a9bcc)、Green (#788c5d)
- 字体：Poppins (标题)、Lora (正文)

## 3. canvas-design

**描述**：使用设计理念在 .png 和 .pdf 文档中创建精美的视觉艺术。

**功能**：
- 创建设计哲学（.md 文件）
- 生成视觉艺术作品（.png 或 .pdf 文件）

**使用场景**：
- 用户请求创建海报、艺术作品、设计或其他静态作品
- 需要原创视觉设计

**实现方式**：
- 两步流程：先创建设计哲学，再实现视觉设计
- 注重空间、形式、颜色、材料、比例和节奏
- 强调工艺和专业设计

## 4. doc-coauthoring

**描述**：引导用户完成协作编写文档的结构化工作流程。

**功能**：
- 上下文收集
- 迭代构建和编辑文档
- 读者测试

**使用场景**：
- 用户想要编写文档、提案、技术规范、决策文档或类似的结构化内容
- 需要高效转移上下文、通过迭代完善内容并验证文档对读者有效

**实现方式**：
- 三阶段工作流：上下文收集、细化与结构化、读者测试
- 引导用户提供元上下文和详细信息
- 逐节构建文档
- 使用新鲜 Claude 实例测试文档

## 5. docx

**描述**：全面的文档创建、编辑和分析，支持跟踪更改、评论、格式保留和文本提取。

**功能**：
- 读取和分析文档内容
- 创建新 Word 文档
- 编辑现有文档
- 支持跟踪更改

**使用场景**：
- 需要处理专业文档 (.docx 文件)
- 创建新文档
- 修改或编辑内容
- 处理跟踪更改
- 添加评论

**实现方式**：
- 使用 pandoc 进行文本提取
- 使用 ooxml 脚本进行原始 XML 访问
- 使用 docx-js 创建新文档
- 支持红lining 工作流处理现有文档

## 6. frontend-design

**描述**：创建具有高质量设计的独特、生产级前端界面。

**功能**：
- 设计思考和审美方向选择
- 实现 HTML/CSS/JS、React 等代码
- 确保视觉吸引力和记忆点

**使用场景**：
- 用户请求构建网页组件、页面、工件、海报或应用程序
- 需要创建网站、登录页面、仪表板、React 组件、HTML/CSS 布局
- 需要美化任何 Web UI

**实现方式**：
- 设计思考：理解目的、选择审美方向、考虑约束
- 实现生产级代码
- 注重排版、颜色与主题、动画、空间构图、背景与视觉细节

## 7. internal-comms

**描述**：一套资源，用于帮助编写各种内部通信，使用公司喜欢的格式。

**功能**：
- 提供各种内部通信模板
- 指导通信格式和内容

**使用场景**：
- 用户需要编写内部通信
- 包括状态报告、领导更新、3P 更新、公司通讯、常见问题解答等

**实现方式**：
- 根据通信类型选择适当的指南文件
- 从 `examples/` 目录加载相关文件：
  - `3p-updates.md` - 用于进度/计划/问题团队更新
  - `company-newsletter.md` - 用于公司范围内的通讯
  - `faq-answers.md` - 用于回答常见问题
  - `general-comms.md` - 用于其他通信类型

## 8. kotlin-ecs

**描述**：用于 Sect 项目编写 Kotlin ECS（实体组件系统）代码的指南。

**功能**：
- 提供 ECS 架构指导
- 代码风格和最佳实践
- 组件、系统、查询等实现指南

**使用场景**：
- 为 Sect 项目编写或修改 Kotlin ECS 代码
- 包括组件、系统、实体和查询

**实现方式**：
- 基于 `lko-ecs` 模块的核心 ECS 实现

- 提供组件、系统、Addon、事件和查询系统的详细指南

## 9. mcp-builder

**描述**：创建高质量 MCP（模型上下文协议）服务器的指南，使 LLM 能够通过精心设计的工具与外部服务交互。

**功能**：
- 指导 MCP 服务器开发
- 提供 API 设计最佳实践
- 支持 Python（FastMCP）或 Node/TypeScript（MCP SDK）

**使用场景**：
- 构建 MCP 服务器以集成外部 API 或服务
- 需要创建支持 LLM 与外部服务交互的工具

**实现方式**：
- 四个主要阶段：深入研究和规划、实现、审查和测试、创建评估
- 推荐使用 TypeScript 作为开发语言
- 提供完整的开发工作流程和最佳实践

## 10. pdf

**描述**：全面的 PDF 处理工具包，用于提取文本和表格、创建新 PDF、合并/拆分文档以及处理表单。

**功能**：
- 读取和分析 PDF 内容
- 提取文本和表格
- 创建新 PDF
- 合并和拆分 PDF
- 处理 PDF 表单
- 支持 OCR 扫描的 PDF

**使用场景**：
- 需要填写 PDF 表单
- 需要以编程方式处理、生成或大规模分析 PDF 文档

**实现方式**：
- 使用 Python 库：pypdf、pdfplumber、reportlab
- 使用命令行工具：pdftotext、qpdf、pdftk
- 支持各种 PDF 操作和转换

**描述**：全面的电子表格创建、编辑和分析，支持公式、格式、数据分析和可视化。

**功能**：
- 读取和分析电子表格内容

- 数据分析和可视化

**使用场景**：

## 技能使用方法

所有技能都可以通过以下方式调用：
```bash
openskills read <skill-name>
```

技能内容将加载到上下文中，并提供详细的使用说明和资源路径。

## 技能目录结构

所有技能都遵循以下目录结构：
```
skill-name/
├── SKILL.md (必需) - 技能元数据和说明
└── 可选资源
    ├── scripts/          - 可执行代码
    ├── references/       - 参考文档
    └── assets/           - 输出中使用的文件
```
